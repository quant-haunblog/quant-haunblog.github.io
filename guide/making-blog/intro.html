VitePress 블로그 만들기 ​

Language

  • English Page: 영어 페이지

퀀트하운블로그 만든 과정 ​

블로그 만든 과정

  1. 정적 사이트 생성기인 VitePress 를 이용하여 정적 사이트(퀀트하운블로그)를 만든다.
  2. Github Pages 로 블로그를 게시할 인터넷 주소(URL)을 만든다.
  3. 배포(Deploy) 기능을 통해 블로그를 Github 저장소에 배포한다. 이제 인터넷에서 블로그가 보인다.
  4. algolia 에 사이트를 등록하여 검색기능을 구현한다.
  5. Google Search Console 에 블로그 sitemap과 URL 을 등록하여 구글 검색엔진에 내 글들을 등록한다.
  6. Google Analytics 에 블로그를 연동하여 방문자들을 분석한다. 어떤 키워드로 검색해서 들어왔는지 볼 수 있다.
  7. Google Adsense 에 광고 게재 신청하여 승인받는다. 승인 후 내가 원하는 위치에 광고를 게재한다.
  • 위의 과정마다 하나의 주제로 페이지 작성하여 소개해 드리겠습니다.

개인 블로그를 직접 만든 이유 ​

개인 블로그를 직접 만든 이유

  • 개인이 인터넷 공간에서 글을 쓰는 일반적인 방법으로 다양한 블로그 플랫폼(네이버블로그, 티스토리, 워드프레스, 벨로그 등)이 있습니다.

  • 저 같은 개발 관련 지식이 없는 직장인은 블로그 플랫폼을 사용하는 것이 오롯이 글 작성에만 집중하는 방법일 수 있습니다.

  • 그러다 정적 사이트 생성기(SSG)라는 기술을 활용하면 빠르고 콘텐츠 중심의 정적 웹사이트를 직접 구축할 수 있다는 것을 알게 되었습니다.

  • 새로운 기술에 대한 궁금증도 있고, 어차피 화려한 썸네일의 예쁜 블로그가 아닌 심플하고 가벼운 글 위주의 개인 블로그를 만들고자 했기에 블로그 플랫폼이 아닌 직접 정적 사이트를 만들어 글을 쓰게 되었습니다.

  • 이 블로그를 만들고 다듬어 나가는 과정 중에 배운 것들이 많기에 간단히 그 경험을 공유합니다.

개인이 직접 블로그를 만드는 방법 ​

개인이 직접 블로그를 만드는 방법

  • 개인이 직접 블로그를 만들어 인터넷 공간에 보여줄려면(이를 호스팅이라고 함),
    1. 호스팅할려면 블로그 (다양한 방법으로 블로그 웹사이트를 만들 수 있음)
    2. 블로그를 호스팅할 서버 (호스팅 서비스를 제공하는 다양한 방법이 존재)
  • 위의 두 가지가 필요합니다.

호스팅할려는 블로그 만드는 방법 ​

웹사이트 형태 선택 ​

웹사이트 형태 선택

  • 먼저 인터넷 공간에 보여줄 블로그를 웹사이트 형태로 만들어야 합니다.
  • 하나의 웹사이트는 여러 개의 HTML 파일로 구성됩니다. 블로그의 경우 글마다 하나의 HTML 파일이 만들어 진다고 생각하면 됩니다.
  • 이런 HTML 파일을 직접 하나씩 만들어야 한다면 글을 제대로 작성할 수 없겠죠.
  • 그래서 웹사이트를 만드는 다양한 방법이 있고, 만들고자 하는 웹페이지의 형태에 맞는 방법으로 개발하는 것이 좋습니다.
  • 퀀트하운블로그는 글 위주의 심플한 블로그이기 때문에 미리 HTML 파일을 만들어 방문자에게 항상 같은 내용을 보여주면 되고, 별도의 상호작용이 필요하지 않으므로 정적 사이트 (Static Site) 형태로 선택하였습니다.

정적 사이트란? ​

정적 사이트

  • 정적 사이트 (Static Site)는 HTML, CSS, JavaScript 로만 이루어진 사이트 입니다.
  • 보통 동적인 사이트들은 사용자의 액션에 따라 웹페이지가 바뀌어 가지만, 정적인 사이트는 사용자와 상호액션 없이 항상 같은 내용을 보여주기 때문에 구축이 쉽고 속도가 빠르다는 장점이 있습니다.

정적 사이트 생성기란? ​

정적 사이트 생성기(Static Site Generator)

  • 정적 사이트 생성기(Static Site Generator)는 정적 사이트를 쉽게 작성하기 위한 프레임워크를 말합니다.
  • 정적 사이트 생성기는 정해진 틀에 따라 폴더, md파일, 이미지 등등을 배치하면, 이를 조합해 미리 HTML 파일을 만들어 두고 서버가 이를 호스팅하기만 하면 되도록 사이트를 생성해 줍니다.
  • 각각의 개발 언어에 기반한 다양한 정적 사이트 생성기가 존재합니다. 보통 많이 사용하는 Jekyll 은 Ruby 기반으로 만들어져 있고 Gatsby 는 React 기반으로 만들어져 있습니다.
  • 많이 사용하지는 않지만 Vue 를 기반으로 한 VuePress 와 VitePress 도 있습니다. 둘 다 심플한 웹사이트 구축에 용이한대 Vue 에서 VitePress 를 계속 지원할 예정이기에 VitePress 로 블로그를 작성하게 되었습니다.
  • VitePress 로 정적 사이트를 만드는 방법은 다음 글에서 소개합니다.

사이트를 호스팅할 서버 ​

호스팅 서버

  • 정적 사이트 생성기로 사이트를 만들었다면, 이 사이트를 호스팅할 서버가 필요합니다.
  • 집에 직접 서버컴퓨터를 두고 호스팅할 수도 있고, 유료 호스팅 업체를 통할수도 있겠지만 크기가 작은 개인블로그 하나 정도는 무료로도 호스팅할 수 있습니다.
  • 무료로 호스팅 서비스를 제공하는 곳 중 Github Pages 가 있습니다.

Github Pages

  • Github Pages 는 깃허브아이디만 있으면 사이트 하나를 무료로 호스팅 해줍니다.
  • Github에 저장소(레포지토리)를 만들고 빌드된 사이트를 올리면, [깃허브아이디].github.io 를 통해 접근할 수 있습니다.
    • 퀀트하운블로그의 주소인 quant-haunblog.github.io 에서 quant-haunblog 이 저의 깃허브아이디 입니다.
  • 딱히 뭔가 설치하지 않아도 되고 트레픽 용량도 개인 블로그 크기에 비해 충분하기에 상당히 좋은 서비스입니다.
  • Github Pages 에서는 정적 생성기로 jekyll 를 기본 제공합니다. jekyll + Github Pages 로 사이트를 만들면 글을 추가해 github에 올리기만 해도 github에서 자동으로 사이트를 생성해주기 때문에 개인블로그를 만들 때 이 조합으로 하면 편리합니다.
  • 하지만 다른 정적 사이트 생성기를 사용해 웹사이트를 만들더라도 무료 호스팅 서비스를 사용할 수 있기 때문에 본인이 원하는 정적 사이트 생성기를 사용해도 무방합니다.
  • 퀀트하운블로그는 VitePress 로 웹사이트를 만들고 Github Pages 로 호스팅하고 있습니다.
  • Github Pages 에서 도메인을 만들고 배포하는 과정도 다음 글에서 소개합니다.
