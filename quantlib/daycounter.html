퀀트립(QuantLib-Python): DayCounter 클래스 ​

QuantLib DayCounter 클래스는 금융 시장에서 날짜 계산을 위해 사용되는 다양한 방법을 구현한 클래스입니다. 이 클래스는 두 날짜 사이의 일수를 계산하거나, 특정 날짜 간의 비율을 계산하는 데 사용됩니다.

금융 계약에서는 이자 계산 방식이 중요하기 때문에, DayCounter 클래스는 다양한 일수 계산 규칙(예: Actual/Actual, Actual/360, 30/360 등)을 제공합니다. 이를 통해 정확한 이자 계산이 가능해지고, 서로 다른 금융 상품 간의 일관성을 유지할 수 있습니다.

DayCounter 클래스 개요 ​

DayCounter Class

  • 금융 거래에 있어서 시장 참여자들 모두가 공통적으로 합의한 규칙이 있고 FICC(Fixed Income, Currency, Commodity) 시장에서는 정말 다양한 시장관행들(Market Conventions)이 있다.

  • FICC 상품을 프라이상하는 것에서 가장 중요한 부분은 이러한 여러 가지 시장관행들에 대한 디테일을 어떻게 구현에 적절히 반영할 것인가이다.

  • 일자 계산방식 또한 휴일과 마찬가지로 국가 그리고 시장마다 적용하는 관행이 다른데, QuantLib에서는 DayCounter라는 클래스를 통해 다양한 일자 계산방식을 제공한다.

일자 계산방식

  • 일자 계산방식은 이자 계산 구간이 1년 미만인 경우에 이자를 어떻게 계산해야 할 것인가에 대한 상호 간의 약속이다. 이것은 3개월 혹은 6개월 단위로 이자 교환을 하는 스왑시장과 채권시장에서 가장 중요한 디테일이다.

  • 금리의 고시 단위가 연율이기 때문에, 1년 미만 기간의 이자를 계산하기 위해서는 기본적으로 시간 프레임을 연단위로 맞출 필요가 있다.

  • 일자 계산방식은 쉽게 말해 두 가지를 정하면 되는데, 하나는 1년을 몇일로 볼 것인가, 다른 하나는 한달을 몇일로 볼 것인가이다.

  • 우선 1년을 몇일로 볼 것인지에 대해서는 ACT, 365, 360, 252가 있다.

    • 여기서 ACT란 Actual 의 약자로 실제 그 해의 일수를 계산하는 방식이다. 만약 ACT를 쓴다면 윤년의 경우 1년이 366일이 될 것이다.
    • 나머지 관행들은 1년을 365일, 360일, 252일로 고정해서 쓰는 방식이다.

  • 한달을 몇일로 볼 것인가는 ACT, 30, 영업일이 있다.

    • ACT 는 실제 이자 구간의 일수를 계산하는 방식이며, 30 은 한 달을 30일로 보겠다는 방식이다.
    • 마지막으로 영업일은 휴일을 제외한 영업일만을 계산하겠다는 방식이다.
    • 예를 들어, 8월 2일부터 10월 2일까지의 구간에 대해 30 방식을 사용한다면 2개월과 1일의 기간이므로 30 X 2 + 1 = 61일로 계산하게 된다.

  • 두 요소를 최종적으로 조합하면 ACT/365, ACT/360, ACT/ACT, 30/360 등과 같이 표현할 수 있으며, 이들이 바로 이자 일수 계산을 위한 시장 관행이다.

    • 분자에는 한 달 관행이, 분모에는 1년 관행이 위치한다.
    • 국가마다 사용하는 관행이 서로 다른데, 미국은 ACT/360, 유럽은 30/360, 한국은 ACT/365 방식을 사용하고 있다.

DayCounter 클래스 구조 ​

DayCounter 클래스 구조

  • DayCounter 클래스를 구현하는 방법은 아주 간단하다. Calendar 클래스와 마찬가지로 사용하고자 하는 방식을 단순히 선언만 해주면 된다. 아래의 코드는 각각의 일자 계산방식을 나타낸 예시이다.
python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

Day Count Convention

  • Actual360 : Actual / 360

  • Actual365Fixed : Actual / 365(Fixed)

    • Standard
    • Canadian
    • NoLeap

  • ActualActual : Actual / Actual

    • ISMA
    • Bond
    • ISDA
    • Historical
    • Actual365
    • AFB
    • Euro

  • Business252 : Business / 252

  • Thirty365 : 30 / 360

    • USA
    • BondBasis
    • European
    • EurobondBasis

  • Italian

    • German
    • ISMA
    • ISDA

  • SimpleDayCounter : NASD

DayCounter 클래스 기능 ​

DayCounter 클래스 기능

  • DayCounter 클래스는 크게 두 가지 기능을 가지고 있다.
  • 하나는 두 날짜 사이의 일자를 계산하는 dayCount() 메서드이며, 다른 하나는 시장 관행을 기반으로 두 날짜 사이의 기간을 연단위로 계산해 주는 yearFraction() 메서드이다.

dayCount() ​

dayCount

  • dayCount() 메서드는 주어진 일자 계산방식을 바탕으로 두 날짜 사이의 기간을 일할로 계산하는 기능을 제공한다.
python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

# Function
startDate = ql.Date(15, 5, 2024)
endDate = ql.Date(15, 8, 2024)

# dayCount(Date1, Date2)
print(f'Day Count by Actual/360 = {act360.dayCount(startDate, endDate)}')
print(f'Day Count by Actual/365 = {act365.dayCount(startDate, endDate)}')
print(f'Day Count by Actual/Actual = {actact.dayCount(startDate, endDate)}')
print(f'Day Count by 30/360 = {thirty360.dayCount(startDate, endDate)}')
print(f'Day Count by BusinessDay/252 = {b252.dayCount(startDate, endDate)}')
print(f'Day Count by SimpleDayCount = {simpledc.dayCount(startDate, endDate)}')

## Output
## Day Count by Actual/360 = 92
## Day Count by Actual/365 = 92
## Day Count by Actual/Actual = 92
## Day Count by 30/360 = 90
## Day Count by BusinessDay/252 = 65
## Day Count by SimpleDayCount = 90

yearFraction() ​

yearFraction

  • yearFraction() 메서드는 특정한 일자 계산방식을 사용했을 때 주어진 두 날짜 사이의 기간을 연단위로 환산하는 기능을 제공한다.
  • 각각의 방식은 1년을 몇일로 계산할 것인가에 따라 각각 다른 결과값을 출력해 준다. 다시 말해 시장 관행 방식을 어떻게 사용하느냐에 따라 계산된 이자금액이 서로 다르다는 의미이다.
python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

# Function
startDate = ql.Date(15, 5, 2024)
endDate = ql.Date(15, 8, 2024)

# yearFraction(Date1, Date2)
print(f'Year Fraction by Actual/360 = {round(act360.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by Actual/365 = {round(act365.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by Actual/Actual = {round(actact.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by 30/360 = {round(thirty360.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by BusinessDay/252 = {round(b252.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by SimpleDayCount = {round(simpledc.yearFraction(startDate, endDate), 4)}')

## Output
## Year Fraction by Actual/360 = 0.2556
## Year Fraction by Actual/365 = 0.2521
## Year Fraction by Actual/Actual = 0.2514
## Year Fraction by 30/360 = 0.25
## Year Fraction by BusinessDay/252 = 0.2579
## Year Fraction by SimpleDayCount = 0.25
