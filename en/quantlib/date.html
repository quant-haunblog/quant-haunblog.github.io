QuantLib-Python: Date Class ​

The QuantLib Date class is used to handle important date calculations in financial markets.

Dates are a crucial element in financial engineering for tasks such as derivative pricing, interest calculation, and determining maturity dates, and this class is designed to efficiently handle a wide range of date-related operations.

Overview of the Date Class ​

Overview of the Date Class

  • Before evaluating the price of a derivative, the concept of time must be established. This is because as time progresses, the underlying asset and other external variables change, and these changes affect the price of the derivative.
  • Even if all variables remain constant, the price of a derivative will change over time due to the presence of an expiration date. The passage of time inherently influences the price of derivatives.
  • The moments of birth and expiration for a derivative are defined as the Effective Date and Maturity Date. As each day passes from the effective date to the maturity date, the price of the derivative changes.
  • Therefore, it is essential in the world of derivatives to specify dates like the effective date, maturity date, or settlement date. To facilitate this, QuantLib provides the Date class, which is used to declare and manage these critical dates.

Structure of the Date Class ​

Date Class Implementation Method #1 ​

Method #1: ql.Date(Day, Month, Year)

python
import QuantLib as ql

# ql.Date(Day, Month, Year)
date = ql.Date(30, 6, 2024)
print(date.ISO())  # 2024-06-30
  • This method involves inputting Day, Month, and Year as parameters.
  • It is widely used due to its simplicity and intuitive nature.

Date Class Implementation Method #2 ​

#2, ql.Date(dateString, formatString)

python
import QuantLib as ql

# ql.Date(dateString, formatString)
date = ql.Date('20240630', '%Y%m%d')
print(date.ISO())  # 2024-06-30
  • This method involves inputting date information in the form of a string.
  • In practical scenarios, especially in financial companies, this method is more convenient than Method #1 when reading and using date information directly from the company’s database.

Date Class Implementation Method #3 ​

#3, # ql.Date(serialNumber)

python
import QuantLib as ql

# ql.Date(serialNumber)
date = ql.Date(45473)
print(date.ISO())  # 2024-06-30
  • This method is similar to how dates are handled in Excel, where January 1, 1900, is set as 1, and each subsequent day is counted sequentially.
python
import QuantLib as ql

# Construction
date1 = ql.Date(30, 6, 2024)
date2 = ql.Date('20240630', '%Y%m%d')
date3 = ql.Date(45473)

print(date1)  # 2024-06-30
print(date2)  # 2024-06-30
print(date3)  # 2024-06-30

Functions of the Date Class ​

Key Features of the Date Class ​

1. Date Creation and Manipulation

  • You can create a date object with a specific year, month, and day.
  • You can perform operations like adding or subtracting days from a date. For example, you can calculate the date 30 days or 1 year from a given date.

2. Day of the Week Calculation

  • You can check what day of the week a specific date falls on.

3. Business Day Calculation

  • Calculating business days is crucial in financial markets. This class can be used in conjunction with various financial calendars to check whether a specific date is a business day or to adjust dates based on business day conventions.

4. Date Comparison

  • You can compare two dates to calculate the difference between them or check whether one date is earlier or later than another.

Member functions ​

The Date class provides information about a specific date, such as the day of the week, month, and year.

python
date = ql.Date(30, 6, 2024)  # 2024-06-30

# ISO() : ISO format, YYYY-MM-DD
print('ISO format:', date.ISO())  # ISO format: 2024-06-30

# weekday()
# an integer that returns the number corresponding to the week
# Sunday:1 - ... - Saturday:7
print('Weekday:', date.weekday())  # Weekday: 1

# dayOfMonth()
# integer, the date returned is the day of the month
print('Day of Month:', date.dayOfMonth())  # Day of Month: 30

# dayOfYear()
# integer, the date returned is the day of the year
print('Day of Year:', date.dayOfYear())  # Day of Year: 182

# month()
# an integer that returns the month corresponding to the date
print('Month:', date.month())  # Month: 6

# year()
# an integer that returns the year corresponding to the date
print('Year:', date.year())  # Year: 2024

# serialNumber()
# integer, the number of days corresponding to the return date 
# (starting from 1899-12-31)
print('Serial Number:', date.serialNumber())  # Serial Number: 45473

Static functions ​

python
# Date.todaysDate()
# Date object, which returns the current date of the system.
print('Today:', ql.Date.todaysDate())  # Today: September 5th, 2024

# Date.minDate()
# Date object, which returns the minimum date that QuantLib can represent.
print('Min Date:', ql.Date.minDate())  # Min Date: January 1st, 1901

# Date.maxDate()
# Date object, which returns the maximum date that QuantLib can represent.
print('Max Date:', ql.Date.maxDate())  # Max Date: December 31st, 2199

# Date.isLeap(y)
# Boolean value to determine whether y is a leap year
print('Is Leap:', ql.Date.isLeap(2024))  # Is Leap: True

# Date.endOfMonth(d)
# Date object, which returns the date corresponding to the end of the month where the date d is located
print('End of Month:', ql.Date.endOfMonth(ql.Date(4, 8, 2024)))  # End of Month: August 31st, 2024

# Date.isEndOfMonth(d)
# Boolean value to determine whether d is at the end of the month
print('Is Month End:', ql.Date.isEndOfMonth(ql.Date(29, 9, 2024)))  # Is Month End: False
print('Is Month End:', ql.Date.isEndOfMonth(ql.Date(30, 9, 2024)))  # Is Month End: True

# Date.nextWeekday(d, w)
# Date object, which returns the date corresponding to the first week w after date d 
# (for example, the first Friday after 2018-03-12)
print('Next WD:', ql.Date.nextWeekday(ql.Date(1, 9, 2024), ql.Friday))  # Next WD: September 6th, 2024

# Date.nthWeekday(n, w, m, y)
# Date object, which returns the date corresponding to the n week w in the given month m and year y 
# (for example, the third Wednesday of July 2010)
print('n-th WD:', ql.Date.nthWeekday(3, ql.Wednesday, 9, 2024))  # n-th WD: September 18th, 2024
